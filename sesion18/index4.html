GET

  • se utiliza para solicitar datos del servidor.
  • los datos del formulario se agregan a la URL como parametros de consulta
  • es visible en la barra de direcciones del navegador y puede ser macado o guardado en el navegador.
  • Es adecuado para solicitudes que no cambian el estado del servidor (consultad de bases de datosy busqueda)
  • Los datos tienen una longitud limitada por las restricciones de longitud URL

POST

  • Envia datos al servidor para procesarlo datos del servidor.
  • los datos del formulario no son visibles
  • es adecuado para enviar datos confidenciales y ademas pueden ser mas extensos.
  • No se utiliza para solicitudes que cambian el estado del servido de ser asi se debe usar con precaucion
  • No tiene restriccion de longitud como el metodo GET
